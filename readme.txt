	TIDEngine - version 1.0 beta&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; <img style="width: 128px; height: 128px; float: right;" src="http://siteinfoservice.com/data/uploads/TIDEngine-128-blue.gif" alt="TIDEngine Logo 256x256"></h1>
<h3>
	<strong>TIDEngine </strong><strong>is</strong> PHP Based Template Engine/Sysytem&nbsp; build for Content Management System with codename <strong>"</strong><strong>TIDEngine</strong><strong>"</strong>.</h3>
<p>
	As <strong>"</strong><strong>TIDEngine</strong><strong>"</strong> is still under heavy development by <em><strong>Site Info Service Development Team</strong></em> and will not be published for some time, <strong>TIDEngine Template </strong><strong>System </strong>was born as separate component.</p>
<h2>
	<strong>TIDEngine CMS</strong></h2>
<p>
	<strong>TIDEngine CMS </strong>is based on&nbsp; <strong>MVC Model</strong> and <strong>Singleton Design Patern </strong> width key concepts</p>
<ul>
	<li>
		SEO and Performance Optimization</li>
	<li>
		Friendly User Interface</li>
	<li>
		Easy Plug-in and Modules System</li>
	<li>
		Control over every Web Page Element</li>
	<li>
		Easy Tempating</li>
</ul>
<p>
	By default Friendly URLS's , posibility to manipulate with most important SEO parameters from Admin Panel. Cross Browser Optimization...</p>
<p>
	&nbsp;</p>
<p>
	As most other Template Engines, TIDEngine Template System have one purpose to seperate code and layout. TIDEngine Template System take power of PHP for complete code processing, so there are no new syntax to learn. Also TIDEngine do not mix coding with design at all, means there are no loops, conditions...<br>
	That is PHP job to be done. There are also build in powerfull Caching Engine with Javascript , CSS Optimization, Combining and Compressing. This Caching Engine gives high speed to your Web Page, lower number of HTTP requests and gives quality experience to site visitor.</p>
<p>
	<strong style="color: rgb(0, 51, 153);">You just need to include reviously defined {SHORTCODE} in yours template files. </strong></p>
<p>
	You can use this Template System as standalone component or to incorporate in your projects. Send data to Engine and Generate Web Page. More info you can find in other selections.</p>

<h2>
	<img style="width: 32px; height: 32px;" src="http://siteinfoservice.com/data/uploads/TIDEngine-32-blue.gif" alt="TIDEngine Logo">&nbsp; FEATURES:</h2>
<ol>
	<li>
		<strong>Optimize Javascript files with one of included Javascript packers:</strong>
		<ul>
			<li>
				Packer</li>
			<li>
				JSmin</li>
			<li>
				JShrink</li>
			<li>
				JSminPlus</li>
			<li>
				JSminPlus</li>
		</ul>
		<p>
			Use native Javascript and Framework minified version of files. If they do not exist TIDEngine create them.<br>
			You should expet High Server load so please use it only once and set expire headers in far future, or them on local machine.</p>
	</li>
	<li>
		<strong>Optimize CSS files with native TIDEngine CSS Files Optimizer not CSS Cleaner just removing whitespace and comments.</strong></li>
	<li>
		Reduces the number of HTTP requests:
		<ul>
			<li>
				combinig all Javascript Files in one or two files (native Javascript and Framework).</li>
			<li>
				combinig all CSS Files in one file.</li>
		</ul>
	</li>
	<li>
		<strong>Reduces the size of the HTTP response by compressing all componets with gzip </strong>(Javacript, CSS and Template Files).</li>
	<li>
		<strong>Use Client Caching for all Page Components</strong> (Javacript, CSS and Template Files).</li>
	<li>
		Use Server Caching for all Optimized Page Components (Javacript, CSS and Template Files).</li>
	<li>
		<strong>Page cache versioning and on fly page data changes check</strong>. So if any there are any data changes on page new cache file will be created.</li>
	<li>
		<strong>Cache expiration check.</strong> If one of Cache Files expire new will be created, all other wil be in use untill they expires.</li>
	<li>
		<strong>XHTML Code Optimization.</strong> There are 2 possibilities. You can indent XHTML or remove all whitespace and put complete XHTML in one line.</li>
	<li>
		<strong>Browser detection</strong> TIDEngine and Browscap build in support for native get_browser() function.</li>
</ol>
